# Glossary

This glossary collects the technical terms and definitions used throughout the documentation.

## A

- **Ansible**: Configuration management and deployment automation tool. See [Ansible Base](doc/ansible/ansible_base.md).

## C

- **Container**: Isolated environment that runs an application together with its dependencies. See [Docker Base](doc/docker/docker_base.md).
- **Ceph**: Distributed storage system. See [Ceph Base](doc/storage/ceph/ceph_base.md).

## D

- **Deployment**: Kubernetes resource that manages pod replicas. See [Kubernetes Base](doc/kubernetes/kubernetes_base.md).
- **Docker**: Container platform. See [Docker Base](doc/docker/docker_base.md).
- **Dockerfile**: File containing the instructions used to build Docker images. See [Docker Base](doc/docker/docker_base.md#dockerfile).

## H

- **HAProxy**: Load balancer and reverse proxy. See [HAProxy Base](doc/haproxy/haproxy_base.md).

## I

- **Ingress**: Kubernetes resource used to expose HTTP/HTTPS services. See [Kubernetes Base](doc/kubernetes/kubernetes_base.md).
- **Infrastructure as Code (IaC)**: Managing infrastructure through code. See [Terraform Base](doc/terraform/terraform_base.md).

## K

- **Kubernetes (K8s)**: Container orchestration platform. See [Kubernetes Base](doc/kubernetes/kubernetes_base.md).

## N

- **Namespace**: Kubernetes namespace used to isolate resources. See [Kubernetes Base](doc/kubernetes/kubernetes_base.md).

## O

- **OpenStack**: Cloud computing platform. See [OpenStack Base](doc/openstack/openstack_base.md).
- **OSD**: Object Storage Daemon (Ceph). See [Ceph Base](doc/storage/ceph/ceph_base.md).

## P

- **PG**: Placement Group (Ceph). See [Ceph Base](doc/storage/ceph/ceph_base.md).
- **Pod**: Smallest deployable unit in Kubernetes, holding one or more containers. See [Kubernetes Base](doc/kubernetes/kubernetes_base.md).
- **Proxmox VE**: Virtualization platform. See [Proxmox Base](doc/proxmox/proxmox_base.md).
- **Probe**: Health check in Kubernetes (readiness/liveness). See [Kubernetes Probes](doc/kubernetes/probes.md).

## S

- **Service**: Kubernetes resource that exposes pods internally. See [Kubernetes Base](doc/kubernetes/kubernetes_base.md).

## T

- **Terraform**: IaC tool. See [Terraform Base](doc/terraform/terraform_base.md).
