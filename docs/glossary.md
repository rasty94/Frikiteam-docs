# Glosario

Este glosario recoge términos técnicos y definiciones usadas en la documentación.

## A

- **Ansible**: Herramienta de automatización de configuración y despliegue. Ver [Ansible Base](doc/ansible/ansible_base.md).

## C

- **Container**: Entorno aislado que ejecuta una aplicación y sus dependencias. Ver [Docker Base](doc/docker/docker_base.md).
- **Ceph**: Sistema de almacenamiento distribuido. Ver [Ceph Base](doc/storage/ceph/ceph_base.md).

## D

- **Deployment**: Recurso de Kubernetes para gestionar réplicas de pods. Ver [Kubernetes Base](doc/kubernetes/kubernetes_base.md).
- **Docker**: Plataforma de contenedores. Ver [Docker Base](doc/docker/docker_base.md).
- **Dockerfile**: Archivo con instrucciones para construir imágenes Docker. Ver [Docker Base](doc/docker/docker_base.md#dockerfile).

## H

- **HAProxy**: Balanceador de carga y proxy reverso. Ver [HAProxy Base](doc/haproxy/haproxy_base.md).

## I

- **Ingress**: Recurso de Kubernetes para exponer servicios HTTP/HTTPS. Ver [Kubernetes Base](doc/kubernetes/kubernetes_base.md).
- **Infrastructure as Code (IaC)**: Gestión de infraestructura mediante código. Ver [Terraform Base](doc/terraform/terraform_base.md).

## K

- **Kubernetes (K8s)**: Plataforma de orquestación de contenedores. Ver [Kubernetes Base](doc/kubernetes/kubernetes_base.md).

## N

- **Namespace**: Espacio de nombres en Kubernetes para aislar recursos. Ver [Kubernetes Base](doc/kubernetes/kubernetes_base.md).

## O

- **OpenStack**: Plataforma de cloud computing. Ver [OpenStack Base](doc/openstack/openstack_base.md).
- **OSD**: Object Storage Daemon (Ceph). Ver [Ceph Base](doc/storage/ceph/ceph_base.md).

## P

- **PG**: Placement Group (Ceph). Ver [Ceph Base](doc/storage/ceph/ceph_base.md).
- **Pod**: Unidad más pequeña en Kubernetes, contiene uno o más contenedores. Ver [Kubernetes Base](doc/kubernetes/kubernetes_base.md).
- **Proxmox VE**: Plataforma de virtualización. Ver [Proxmox Base](doc/proxmox/proxmox_base.md).
- **Probe**: Comprobación de salud en Kubernetes (readiness/liveness). Ver [Kubernetes Probes](doc/kubernetes/probes.md).

## S

- **Service**: Recurso de Kubernetes para exponer pods internamente. Ver [Kubernetes Base](doc/kubernetes/kubernetes_base.md).

## T

- **Terraform**: Herramienta de IaC. Ver [Terraform Base](doc/terraform/terraform_base.md).
